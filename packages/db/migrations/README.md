# Migrations

Schema migrations are generated by Drizzle (`pnpm db:generate`). They land in this directory as
timestamped `.sql` files.

The RLS policy migration `0001_enable_rls.sql` is **hand-maintained** and must run AFTER the
schema migration.

## Apply order

```bash
# 1. Push the schema (or run `drizzle-kit migrate` for tracked migrations)
pnpm db:push

# 2. Apply the RLS policies
psql "$DIRECT_URL" -f packages/db/migrations/0001_enable_rls.sql
```

## Adding a new table

1. Add the table definition to `src/schema/<domain>.ts`.
2. Export it from `src/schema/index.ts`.
3. Run `pnpm db:generate` — this produces a new SQL migration file here.
4. Update `0001_enable_rls.sql` to:
   - `alter table public.<name> enable row level security;`
   - Add policies. Default to deny — only authorize the explicit ownership paths.
5. Add an RLS coverage test in `packages/db/__tests__/rls.spec.ts` (deny-by-default check).
